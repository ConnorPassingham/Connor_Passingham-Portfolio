AI Inventory Forecasting Platform

Client-focused machine learning solution for multi-outlet inventory planning

Context

Completed in 2025 as an Auckland University of Technology research and development project for a real client. The project was completed by a five-person team under university supervision.

Role: Team Manager and Data Analyst

Responsibilities

  • Coordinated tasks, team communication and issue resolution
  • Prepared and analysed client sales data
  • Developed and optimised LightGBM forecasting models
  • Built features including lag and rolling-window variables
  • Supported database integration and model retraining
  • Helped design the client-facing forecasting interface

Problem

The client needed a more consistent method for estimating future product demand across multiple stores and reducing the risks associated with overstocking and understocking.

Solution

Developed a multi-horizon forecasting pipeline that cleaned client data, engineered predictive features, trained a LightGBM model and displayed recommended stock levels through a searchable dashboard.

Process steps

  1. Client database
  2. Data preprocessing and refinement
  3. Train and test split
  4. LightGBM model training
  5. Evaluation metrics
  6. Forecast generation
  7. Dashboard delivery

Key project facts

  • Team of five
  • Real client
  • 14-day forecast horizon
  • Multi-outlet data
  • Interactive search and filtering
  • Client-facing output
  • Database connectivity
  • Continuous retraining design

Technology stack

Python LightGBM Pandas NumPy Scikit-learn Flask HTML CSS JavaScript CSV/DB Integration Git

Learning

This project strengthened my understanding of project leadership, data quality, feature engineering, model evaluation, stakeholder accessibility and translating technical outputs into useful business decisions.

NLP Text Classification

Comparing machine learning models for sentiment and emotion classification

Context

A Data Mining and Knowledge Engineering project comparing three classification algorithms across Covid sentiment and emotion datasets.

Contribution

  • Built the Covid raw-data notebook
  • Built the Covid preprocessed-data notebook
  • Implemented preprocessing and model experiments
  • Contributed to analysis, report writing and results packaging

Datasets

  • Corona NLP sentiment dataset
  • Tweet emotion dataset

Models evaluated

  • Multinomial Naïve Bayes
  • Linear Support Vector Machine
  • Feedforward Neural Network

Pipeline

  1. Raw text
  2. Cleaning and tokenisation
  3. Stratified train/test split
  4. TF-IDF vectorisation
  5. Model training
  6. Precision, recall, F1 and confusion-matrix evaluation
  7. Hyperparameter tuning
  8. Stop when no further improvement is found

Techniques applied

  • Lowercasing
  • URL and symbol cleaning
  • Tokenisation
  • Stop-word removal
  • TF-IDF & N-grams
  • L2 regularisation
  • Grid search
  • Learning-rate adjustment
  • Early stopping

Key results

  • Neural Network with L2 regularisation achieved 0.66 overall accuracy on raw Covid data
  • Macro precision: 0.70
  • Macro recall: 0.65
  • Macro F1: 0.67
  • SVM produced stable performance across both datasets
  • Naïve Bayes was the weakest overall model

Key insight

Preprocessing did not consistently improve classification. Removing punctuation, capitalisation and emojis sometimes removed meaningful sentiment signals.

Technologies

Python Pandas NLTK Scikit-learn TensorFlow / Keras TF-IDF Jupyter Notebook Machine Learning NLP
